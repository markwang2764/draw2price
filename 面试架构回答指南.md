> 项目：慧银科技 · MistralAiFactory  
> 定位：面向机加工企业的 AI 端到端工艺分析平台

---

## 一句话项目定位（10秒破冰）

> "这是一个面向机加工企业的 **AI 工艺分析平台**，核心是把零件图纸通过多模态大模型，端到端自动生成工艺方案、G代码、排产计划和报价单，替代工程师大量的手工经验判断。"

## 整体架构分层

```
┌─────────────────────────────────────────────┐
│           前端层  React 18 + Vite            │
│     用户交互 / 拖拽上传 / SSE 流式渲染        │
├─────────────────────────────────────────────┤
│           后端层  FastAPI + uvicorn          │
│     路由编排 / 业务逻辑 / 异步任务管理        │
├───────────────────┬─────────────────────────┤
│     AI 服务层     │       知识库层            │
│  GPT-4o (云端)   │  ChromaDB 向量数据库      │
│  Qwen (国产)     │  sentence-transformers    │
│  Ollama (本地)   │  RAG 检索增强生成          │
├───────────────────┴─────────────────────────┤
│           导出层  ReportLab + G代码          │
│     PDF 工艺卡 / 报价单 / .nc 数控文件        │
└─────────────────────────────────────────────┘
```

## 核心业务数据流

```
用户上传图纸（图片 / PDF）
        ↓
OCR + Vision 识别图纸
（零件尺寸 / 材料 / 公差 / 标注）
        ↓
RAG 检索相关工艺知识
（刀具库 / 工艺路线 / 成本经验）
        ↓
AI 生成工艺路线
（结合公司设备资源约束）
        ↓
逐工序生成 G 代码
（CNC 车 / 铣 / 加工中心）
        ↓
排产计划
（设备排期 + 人员分配 + 交期估算）
        ↓
成本报价
（材料 + 工时 + 管理费15% + 利润18%）
        ↓
导出文件
（PDF 工艺卡 / PDF 报价单 / .nc G代码）
```

## 四大技术亮点（面试加分项）

### 1. 流式 SSE + AI 思考可视化

用户上传图纸后，不是等待一个黑盒结果，而是通过 **Server-Sent Events** 实时看到 AI 每一步的推理过程。

- 后端：`asyncio.create_task + asyncio.shield` + 心跳机制，防止长连接超时
- 前端：`fetch + ReadableStream` 解析事件流，实时渲染思考日志
- 事件类型：`start / thinking / step_complete / complete / error`

### 2. RAG 知识库增强生成

引入 ChromaDB 向量数据库，把公司积累的工艺经验领域知识向量化存储，解决通用大模型在机加工领域专业知识不足的问题。

- 4 大知识分类：刀具库 / 工艺路线 / 工时成本 / 零件特征
- 检索策略：相关性阈值 0.3，最大上下文 2000 字符
- 支持：txt / pdf / docx 文档上传自动入库

### 3. 可插拔的 AI 后端

AI 调用层统一封装，通过环境变量一键切换，业务代码零改动。

```
MISTRAL_BASE_URL=https://oa.api2d.net/v1   # 云端 GPT-4o
MISTRAL_BASE_URL=https://api.siliconflow.cn # 国产 Qwen
MISTRAL_BASE_URL=http://localhost:11434     # 本地 Ollama
```

### 4. 多模态图纸识别

- PDF 图纸：PyMuPDF **2倍分辨率渲染**，取前5页转 PNG 送视觉模型
- 图片图纸：base64 编码直接调用 Vision API
- EasyOCR：补充提取图纸文字标注（中英双语）
- 两路信息融合后进行工艺分析

## 当前不足与改进方向（体现成熟度）

> 主动说出不足，是面试中体现工程师成熟度的关键

| 当前不足 | 改进方向 |
|---------|---------|
| AI 输出不稳定，依赖 Prompt 质量 | 微调领域专用模型（项目已有完整训练方案） |
| 知识库静态，需手动维护 | 接入 ERP/MES 系统，自动同步生产数据 |
| 单机部署，无横向扩展能力 | 拆分微服务，AI 推理独立成服务，支持水平扩展 |
| G代码未经机床仿真验证 | 接入 Vericut 等仿真软件，形成验证闭环 |
| 报价依赖规则，缺乏历史数据校准 | 引入历史订单数据，训练报价预测模型 |

## 未来规划（端到端大模型训练方案）

项目已规划下一阶段的专用大模型训练路线：

```
输入端：2D图纸 / 3D CAD / 自然语言 / 点云
   ↓
编码层：Swin Transformer V2（视觉）
        UV-Net / BRepNet（CAD）
        PointNet++ / PCT（点云）
        BERT（文本）
   ↓
基座：Qwen2.5-7B / Llama3-8B + LoRA/QLoRA 微调
   ↓
输出端：工艺路线 / G代码 / 工时 / 报价 / 排产
```
