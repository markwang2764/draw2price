# 本地训练待办清单（换电脑后从这里开始）

> 用途：在另一台带 NVIDIA GPU 的电脑上，对本平台做**纯文本 LoRA 微调**。
> 本文件跟随 repo 走，记录"要训什么、数据从哪来、跑什么命令"。
> 全景理想方案见 `../端到端机加工大模型训练方案.md`（那是 8×A100 的工业版，本清单是可落地的精简版）。
> 创建于 2026-06-19（在 Mac 上规划，训练在 N 卡机器执行）。

## 0. 现实约束（务必先读）

- **当前开发机是 Mac（darwin），无 CUDA**，不能本地训练。训练必须在 NVIDIA GPU 机器或云 GPU 上做。
- 文档全景版的"几何预训练 / 多模态对齐 / RLHF"是百万级数据 + 8×A100 方案，**本次不做**。
- 本次只做 **Qwen2.5-7B 的 QLoRA 文本微调**，覆盖 4 个文本任务：
  工艺路线生成 / G代码生成 / 工时估算 / 报价。**不碰视觉**（图纸识别继续用现成多模态 API）。
- 判断原则：**数据决定上限，不是算力**。先攒高质量数据，训练只是一条命令。

## 1. 要训练的目标（任务定义）

把现有 6 步流水线里 4 个**纯文本**步骤的能力，蒸馏/微调进一个本地小模型，替代或兜底云端 API：

| 任务 | 对应现有方法（backend/app/services/mistral_service.py） | 输入 | 输出 |
|------|----------------------------------------------------------|------|------|
| 工艺路线 | `generate_process_plan` | 零件分析 JSON + 公司资源 | 工序列表 JSON |
| G代码 | `generate_gcode` | 单道工序 + 设备 | NC 程序 |
| 工时/排产 | `generate_schedule` | 工艺方案 + 资源 + 数量 | 排产 JSON |
| 报价 | `generate_quotation` | 零件+工艺+排产+资源 | 报价 JSON |

> 图纸识别（`analyze_drawing`）是多模态，**不在本次训练范围**，继续用云端 Vision API。

## 2. 数据从哪来（关键，且能在 Mac 上先做）

本平台自己在产数据，这是最大优势：

1. **运行期产出**：`analysis_service.analyses_cache` 里每次完整分析的结果，
   含 `part_analysis / process_plan / gcode_programs / production_schedule / quotation`。
2. **导出脚本**（待写，见第 5 节 TODO-1）：把这些结果转成 SFT 三元组
   `{system, instruction, input, output}`，格式对齐训练方案文档第 11.2 节。
3. **知识库语料**：`knowledge_service`（ChromaDB）里的工艺规范/刀具库，可作检索增强或额外语料。
4. **质量筛选**：导出的样本需工艺专家/规则过一遍，只留可信的（见 TODO-2）。

目标：先攒 **几百~几千条** 高质量 SFT 样本即可起步（QLoRA 不需要海量数据）。

### SFT 数据格式（LLaMA-Factory alpaca 风格）

```json
{
  "system": "你是专业的机加工工艺工程师AI助手。",
  "instruction": "根据零件分析结果生成加工工艺路线",
  "input": "零件：传动轴，材料45钢，外圆φ50×80，精度IT7，Ra1.6...",
  "output": "{\"steps\":[{\"step_number\":1,\"process_name\":\"下料\",...}]}"
}
```

## 3. 训练环境（在新机器上装）

```bash
# 1. 环境
conda create -n machining-llm python=3.10 -y
conda activate machining-llm

# 2. PyTorch（按机器 CUDA 版本调整，示例 cu121）
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121

# 3. 微调工具栈
pip install "transformers>=4.45" datasets accelerate peft bitsandbytes
pip install llamafactory   # 一站式微调，推荐
# 备选：pip install unsloth  （单卡更省显存更快）
```

## 4. 训练配置（QLoRA，单卡可跑）

基座：`Qwen/Qwen2.5-7B-Instruct`（Apache 2.0，中文强）。
单张 24GB（4090）够跑 7B QLoRA；40GB（A100）更宽裕。

LLaMA-Factory 关键参数（写进 `train_config.yaml`，TODO-3）：

```yaml
model_name_or_path: Qwen/Qwen2.5-7B-Instruct
stage: sft
finetuning_type: lora
quantization_bit: 4            # QLoRA，省显存
lora_rank: 64
lora_alpha: 128
lora_target: q_proj,k_proj,v_proj,o_proj,gate_proj,up_proj,down_proj
dataset: machining_sft         # 指向第2步导出的数据
template: qwen
cutoff_len: 4096
per_device_train_batch_size: 2
gradient_accumulation_steps: 8
learning_rate: 2.0e-4
num_train_epochs: 3
warmup_ratio: 0.1
bf16: true
gradient_checkpointing: true
output_dir: ./machining-llm-lora
```

启动：`llamafactory-cli train train_config.yaml`

## 5. 评估（训完判断有没有用）

对照训练方案文档第 8 节，可自动算的硬指标：

| 指标 | 怎么测 | 目标 |
|------|--------|------|
| G代码语法正确率 | 用 G代码解析器/正则校验 | 100% |
| 工艺工序完整率 | 与专家方案对比工序覆盖 | ≥90% |
| JSON 结构合法率 | `json.loads` 通过率 | 100% |
| 报价 MAPE | 与历史报价对比 | ≤10% |

判定：若微调模型在这些指标上 ≥ 现有 prompt 工程方案，才值得挂回平台。

## 6. 挂回平台（闭环）

训好的 LoRA 合并后用 vLLM 起一个 OpenAI 兼容服务，然后改 `backend/.env`：

```
MISTRAL_BASE_URL=http://localhost:8001/v1
MISTRAL_MODEL=machining-qwen
```

平台 `_call_api` 走 OpenAI 兼容协议，**无需改业务代码**即可切到自训模型。
（注意：`response_format` 结构化输出 vLLM 也支持，沿用即可。）
